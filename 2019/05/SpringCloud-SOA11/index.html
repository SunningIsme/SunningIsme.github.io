服务跟踪

1.服务跟踪

微服务架构微服务增多，一个客户端请求形成一个复杂的分布式服务调用链路，如果任何一个服务延迟过高或错误，都会引起请求失败。
所以SpringCloud提供了Spring Cloud Sleuth这个组件来追踪服务

1.1 Spring Cloud Sleuth(链路监控)

首先引入依赖

然后发送一个请求。
在日志信息中可以发现

第一个参数是服务名字
第二个参数是traceID是一条链路的唯一值。
第三个参数是spanId一条链路里面可以包含很多条spanID，可以理解为一个基本的工作单元。没产生一个工作单元，就会变个值
第四个参数是是否把信息收集并发送出去如果为true。就代表，可以把信息发送出去

这里需要注意几个点

第三：我遇到的一个坑。搞了我挺久的。因为 查看控制台的日志，发现 最后一个参数是true并且是以web发送出去消息的。但是为什么通过zipkin就是看不到服务呢？
