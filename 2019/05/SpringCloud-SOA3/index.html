1.4 @GetMapping @PostMapping

@GetMapping用于将HTTP get请求映射到特定处理程序的方法注解
具体来说，@GetMapping是一个组合注解，是@RequestMapping(method = RequestMethod.GET)的缩写。
@PostMapping用于将HTTP post请求映射到特定处理程序的方法注解
具体来说，@PostMapping是一个组合注解，是@RequestMapping(method = RequestMethod.POST)的缩写。

1.5 @Valid注解和BindingResult bindingResult

作用：用于验证注解是否符合要求，直接加在变量user之前，在变量中添加验证信息的要求，当不符合要求时就会在方法中返回message 的错误提示信息
@Validated和BindingResult bindingResult是配对出现，并且形参顺序是固定的（一前一后）。

我这里用的是@NotEmpty这个注解，用来判断从前台传过来的数据是否为空
看代码就可以大致理解他们的作用了。

1.7 VO,DTO的概念和用处

VO：视图对象，用于展示层，它的作用是把某个指定页面（或组件）的所有数据封装起来
DTO:数据传输对象，这个概念来源于J2EE的设计模式，原来的目的是为了EJB的分布式应用提供粗粒度的数据实体，以减少分布式调用的次数，从而提高分布式调用的性能和降低网络负载，但在这里，我泛指用于展示层与服务层之间的数据传输对象。

这里有个负载均衡问题。比如有很多地址。负载均衡的策略（随机，轮询）
eureka是客户端发现，属于软负载。也就是客户端向服务器拉取可用的服务器，然后根据负载均衡策略，找到服务器发送数据

2.3 服务端负载均衡器Ribbon

可以自动转化客户端发现的负载均衡的服务RestTemplate和Feign Zuul
还封装了http client 和 Ok client 两种请求端实现，默认使用了对ribbon对eureka服务的负载均衡client
其中@LoadBalanced这个注解就是通过ribbon组建实现的
Ribbon实现软负载均衡有三点：
服务发现：发现以来服务的列表（一句服务的名字把该服务下的实例都找出来）
服务选择规则：依据规则如何从多个策略中选择多个服务
服务监听：也就是检测失效的服务
主要组建
ServerList
IRule
ServerListFilter
主要流程，先通过ServerList找到所有目标地址，通过ServerListFilter来筛选服务，最后通过IRule选择一个实例作为最终目标结果
