1.TCP/IP

TCP/IP协议 把以前的七层协议划分为了四层

数据传输流程:先自上而下，后自下而上处理数据头部

1.1 TCP三次握手

IP协议的主要功能包括寻址和分段。
TCP协议和端口号可以唯一标识一个网络进程
TCP报文头

其中seq是用来顺序传送的， 比如序号字段是107携带了100个字节，然后下一个报文就是从207开始
ACK标识期望收到下一个报文的序号
ACK：确认序号标识
SYN：同步序列号，用于建立连接过程
FIN：finish标识，用于释放连接
TCP/IP是建立的的全双工连接，即A可以给B发送信息，B也可以给A发送信息

为什么要三次握手呢：避免乱序

1.2 首次握手的隐患—SYN超时

起因：Server收到Client的SYN，回复SYN-ACK的时候未收到ACK确认，然后Server不断重试直指超时，Linux默认等待63秒才断开连接
防护措施：SYN队列满后，通过tcp_syncookies参数回发SYN Cookie，若为正常连接则Client会回发SYN Cookie，直接建立连接
通过SYN Cookie建立连接，即使满了也可以通过SYN Cookie建立连接。

1.3 保活机制

向对方发送报货探测报文，如果未收到响应则继续发送，尝试多次达到保活探测次数仍未收到相应则断开连接。

1.4 TIME_WAIT状态

2.UDP

什么叫做面向连接和面向非连接
面向连接：打电话之前回拨号吧，等对方接通后就是建立了连接，然后你们再传递信息，确保信心能正确地传给对方。非面向连接就像发广告。我塞到你的门缝上啊等地方。我不管理收不收得到信息。
UDP特点：
面向非连接
不维护连接状态，支持同时向多个客户端传输相同的消息
数据报文只有8个字节，额外开销较小
吞吐量之受限于数据生成速率，传输速率以及机器性能
劲大努力交付，不保证可靠交付，不需要维护复杂的连接状态表
面向报文，不对应用程序的报文信息进行拆分或者合并
